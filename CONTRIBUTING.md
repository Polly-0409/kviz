# Contributing

Спасибо за интерес к проекту.

## Как предложить изменение

1. Сделайте fork репозитория.
2. Создайте отдельную ветку:
   - `feat/...` для новых возможностей
   - `fix/...` для исправлений
   - `docs/...` для документации
3. Внесите изменения и проверьте проект локально.
4. Откройте Pull Request в `main`.

## Локальная проверка

```bash
node --check src/js/app.js
```

## Рекомендации по коду

- Сохраняйте стиль проекта (2 пробела, LF, UTF-8).
- Не добавляйте зависимости без необходимости.
- Для UI-изменений прикладывайте скриншоты в PR.
- Учитывайте, что это клиентский проект: тексты и поведение должны быть понятны конечному пользователю.

## Коммиты

Используйте короткие и понятные сообщения, например:

- `feat: add combined protocol block`
- `fix: correct multi-zone scoring`
- `docs: update onboarding instructions`
